CSS3之弹性盒布局（一）

​ 在最近的响应式项目中，发现弹性布局是网页布局中不可或缺的。它为盒状（块状）模型增加了灵活性，也改进了块模型。由下图可以看出，弹性盒分为两个轴，一个是主轴，一个是交叉轴，主轴方向可以改变（左右改上下），主轴方向改变后，交叉轴方向也会随之改变。

接下来我们来看看浏览器大概支持情况

由上图可以看出，IE对于弹性盒的支持可以说是呵呵了，其他的浏览器还可以，接下来我们来说说它的基本概念。

  • 1.什么是弹性布局？
官方的定义是，用来为盒模型提供做大的灵活性。任何一个容器都可以指定为弹性布局 
//任何容器
.box1{
    display: flex;
}
//对于行内元素
.box2{
    display: inline-flex;
}
//在chrom、Safari等webkit内核的浏览器，要加上-webkit前缀
.box3{
    display: -webkit-flex;
    display:flex;
}
//需注意：设为flex布局以后，子元素的float、clear和vertical-align属性会失效
  • 2.容器的属性
设置在容器的属性有：

flex-direction

flex-wrap

flex-flow

justify-content

align-items

align-content

2.1 flex-direction属性

flex-direction属性决定主轴的方向（项目的排列方向）

有4个取值

row(默认):主轴为水平方向，左端起点

row-reverse:主轴为水平方向，右端起点

column:主轴为垂直方向，上端起点

column-reverse:主轴为垂直方向，下端起点

2.2 flex-wrap属性

默认情况下，项目都排列在一条线上。flex-wrap表示，如果一条轴线排不下，该如何换行

有3个取值

nowrap(默认)：不换行

wrap：换行，第一行在上方

wrap-reverse:换行，第一行在下方

2.3 flex-flow属性

flex-flow属性时flex-direction属性和flex-wrap的简写形式，默认row nowrap，其取值方式同以上

2.4 justify-content属性

定义项目在主轴上的对齐方式

可能有5个取值，对齐方式与主轴的方向有关。假设主轴从左到右

flex-start（默认）：从左到右

flex-end：右对齐

center：居中对齐

space-between：两端对齐，项目直接的间隔都相等

space-around：每个项目两侧的间隔相等。所以，项目直接按的间隔比项目与边框的间隔大一倍

2.5 align-items属性

align-items属性定义项目在交叉轴上的对齐方式

它有5个取值，对齐方式与交叉轴的方向有关。假设交叉轴从上到下

flex-start：交叉轴起点对齐

flex-end：交叉轴终点对齐

center：交叉轴中点对齐

baseline：项目的第一行文字基线对齐

stretch(默认值)：如果项目未设置高度或者auto,将沾满整个容器的高度

2.6 align-content属性

align-content属性定义了多根轴线的对齐方式。如果项目只有一根轴线，该属性不起作用。

可能有6个取值

flex-start：与交叉轴的起点对齐。

flex-end：与交叉轴的终点对齐。

center：与交叉轴的中点对齐。

space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。

space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。

stretch（默认值）：轴线占满整个交叉轴。

  • 3.项目的属性
设置在项目的属性有：

order

flex-grow

flex-shrink

flex-basis

flex

align-self

3.1 order属性

order属性定义项目的排列顺序。数值越小，排列越靠前，默认为0。

3.2 flex-grow属性

flex-grow属性定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。

如果所有项目的flex-grow属性都为1，则它们将等分剩余空间（如果有的话）。

如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。

3.3 flex-shrink属性

flex-shrink属性定义了项目的缩小比例，默认为1，如果空间不足，该项目缩小。

如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。

如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。

注意！！！负值对该属性无效。

3.4 flex-basis属性

flex-basis属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。它可以设为跟width或height属性一样的值（比如350px），则项目将占据固定空间。

3.5 flex属性

flex属性是flex-grow, flex-shrink 和 flex-basis的简写，默认值为0 1 auto。后两个属性可选。

该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值

3.6 align-self属性

align-self属性允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto，表示继承父元素的align-items属性，如果没有父元素，则等同于stretch.该属性可能取6个值，除了auto，其他都与align-items属性完全一致。
